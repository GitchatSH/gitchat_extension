sidebar-chat.js — Chat View Design

Self-contained chat module. Written from scratch for 300px sidebar.

Public API (interface with explore.js)

// Global object — explore.js calls these
window.SidebarChat = {
  open(conversationId) { ... }, // Push chat view, fetch data
  close() { ... }, // Pop back to list, save draft
  isOpen() { ... }, // Returns boolean
  handleMessage(data) { ... }, // Route chat:* messages from provider
  destroy() { ... }, // Cleanup on webview destroy
};

Internal Architecture

sidebar-chat.js Module Structure
// === STATE (single source of truth) ===
_state = { conversationId, messages[], pinned[], members[],
  currentUser, isGroup, hasMoreOlder, hasMoreAfter,
  loadingOlder, isViewingContext, replyingTo, draft }

// === RENDERING ===
renderHeader() — back btn + avatar + name + search + menu
renderMessages() — group → date sep → message bubbles
renderMessage(msg) — bubble + reactions + attachments + status
appendMessage(msg) — smart scroll (only if at bottom)
renderPinnedBanner() — compact pinned preview
renderInput() — textarea + attach + emoji + send

// === SCROLL SYSTEM ===
attachScrollListener() — RAF-throttled, single listener
  ↳ scrollTop < 200 → loadOlder()
  ↳ distBottom > 300 → showGoDown()
  ↳ distBottom <= 100 → hideGoDown() + markRead()
  ↳ context mode + bottom → loadNewer()

// === FEATURES ===
EmojiPicker — overlay, 73 emojis + search
FloatingBar — hover: react, reply, copy, more
SearchBar — replaces header, results overlay, navigate
PinnedView — full overlay with search
ForwardModal — conversation picker
GroupInfoPanel — members, invite, leave/delete
AttachMenu — photo/document picker, drag-drop, paste
MentionAutocomplete — @trigger, member search

// === MESSAGE HANDLERS ===
handleMessage(data) — routes ALL chat:* types
  chat:init, chat:newMessage, chat:olderMessages
  chat:newerMessages, chat:messageEdited, chat:messageDeleted
  chat:reactionUpdated, chat:readReceipt, chat:typing
  chat:searchResults, chat:uploadComplete, chat:linkPreview
  ... (all 30+ message types)

Data Flow

Message Routing
User clicks conversation in list (explore.js)
  → explore.js: SidebarChat.open(convId)
  → sidebar-chat.js: slide animation, show chat container
  → sidebar-chat.js: doAction("chat:open", { conversationId })
  → explore.ts: loadConversationData(convId)
  → explore.ts: postToWebview({ type: "chat:init", payload: {...} })
  → explore.js: routes to SidebarChat.handleMessage(data)
  → sidebar-chat.js: renderHeader(), renderMessages(), renderInput()

User clicks ← back (sidebar-chat.js)
  → sidebar-chat.js: save draft, cleanup state
  → sidebar-chat.js: SidebarChat.close()
  → explore.js: slide animation back, show chat list
  → explore.js: doAction("chat:close")

Message arrives via WebSocket (explore.ts)
  → explore.ts: check if for active conversation
  → YES: postToWebview({ type: "chat:newMessage", ... })
    → explore.js: SidebarChat.handleMessage(data)
    → sidebar-chat.js: appendMessage() with smart scroll
  → NO: update conversation list badge

Key Difference from Branch Trước

Branch trước (FAILED): Port 40+ functions from chat.js → explore.js. Mixed state. Multiple subagents with different assumptions. State fragmentation. Scroll conflicts.

Branch này (NEW):

  • sidebar-chat.js is a BLACK BOX — explore.js chỉ biết open/close/handleMessage
  • Single state object — _state = { ... } — một source of truth duy nhất
  • Written for 300px — không adapt từ editor panel, viết native cho sidebar
  • Scroll as single unit — 1 listener, 1 function, handles ALL scroll cases
  • Self-contained — sidebar-chat.js không import gì từ explore.js ngoài vscode global
