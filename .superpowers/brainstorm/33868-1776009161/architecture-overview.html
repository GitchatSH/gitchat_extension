GitChat Layout Refactor — Architecture Overview

Telegram-style sidebar chat. Viết mới từ đầu, không port từ chat.js.

File Structure

Code Architecture
src/webviews/
  explore.ts ── Sidebar provider (tabs + chat list + chat view handlers)
  chat.ts ── Editor panel (DISABLED, code kept)
  chat-panel.ts ── Draft management + badge (keep as util)
media/webview/
  shared.js ── vscode API, escapeHtml, timeAgo, doAction
  explore.js ── Sidebar layout: tabs, chat list, navigation
  sidebar-chat.js ── NEW: Chat view (messages, input, scroll, search, emoji...)
  shared.css ── Design tokens
  explore.css ── Sidebar layout styles
  sidebar-chat.css ── NEW: Chat view styles

Navigation Flow

Stack Navigation
stack[0] — Chat List
Managed by: explore.js
• GitChat header (native VS Code)
• Tabs: Inbox | Friends | Channels
• Filter chips
• Conversation list
• New Chat dropdown
• User picker (DM/Group)
⇄
stack[1] — Chat View
Managed by: sidebar-chat.js
• Chat header (← back + name + search + menu)
• Pinned message banner
• Messages (infinite scroll)
• Floating action bar
• Emoji picker + reactions
• Reply/edit/forward
• Attachments + link preview
• In-chat search
• Group info panel
• Input area
explore.js calls SidebarChat.open(convId) → sidebar-chat.js takes over
sidebar-chat.js calls SidebarChat.close() → explore.js resumes

Key Design Decisions

1. sidebar-chat.js is self-contained — owns ALL chat state, scroll, search, emoji. No shared state with explore.js except conversationId.

2. Communication via global functions — SidebarChat.open(id) / SidebarChat.close(). explore.js doesn't touch chat DOM.

3. Provider handles routing — explore.ts routes chat:* messages to chat logic, others to explore logic.

4. State persistence via vscode.setState — sidebar-chat.js saves minimal state for webview restore.

5. Editor panel disabled — chat.ts code kept but not activated. All commands redirect to sidebar.

6. Written from scratch — sidebar-chat.js is NEW code for 300px context. Not ported from chat.js.
