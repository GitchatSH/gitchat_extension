GitChat Layout Refactor — Giải thích bằng tiếng Việt

Tổng quan cách hoạt động của hệ thống chat mới trong sidebar

1. Vấn đề ở branch trước

Trước đây, chat hoạt động trong editor panel (vùng giữa màn hình, chỗ viết code). File chat.js có 4530 dòng code, xử lý mọi thứ: tin nhắn, emoji, scroll, search, pin, v.v.

Khi cần chuyển chat sang sidebar (thanh bên trái ~300px), tao đã copy từng function từ chat.js sang explore.js. Kết quả: code bị phân mảnh, các phần không ăn khớp nhau, gây ra hàng loạt bugs (scroll nhảy, data không load, v.v.).

2. Giải pháp mới

Không copy code cũ. Viết file mới tên sidebar-chat.js từ đầu, thiết kế riêng cho sidebar 300px.

Hệ thống có 3 file JS chạy cùng nhau trong sidebar:

  • shared.js — công cụ chung (gửi message, format text, v.v.)
  • explore.js — quản lý layout sidebar: tabs, danh sách chat, tạo group mới
  • sidebar-chat.js — quản lý MÀN HÌNH CHAT: tin nhắn, gửi/nhận, emoji, scroll, search, pin, v.v.

3. Cách 2 file phối hợp

explore.js ↔ sidebar-chat.js
explore.js
Làm gì:
• Hiển thị danh sách chat (Inbox)
• Hiển thị danh sách bạn bè (Friends)
• Hiển thị kênh (Channels)
• Chuyển tab Inbox/Friends/Channels
• Tạo DM/Group mới
• Filter conversations

KHÔNG làm:
• Không render tin nhắn
• Không xử lý scroll
• Không xử lý emoji/reaction
sidebar-chat.js
Làm gì:
• Hiển thị tin nhắn (bubbles)
• Gửi/nhận tin nhắn
• Emoji picker + reactions
• Scroll vô hạn (load cũ/mới)
• Search trong chat
• Pin messages
• Reply/Edit/Forward/Delete
• Attachments (ảnh, file)
• Group management

KHÔNG làm:
• Không biết về tabs
• Không biết về danh sách chat
Giao tiếp giữa 2 file:
explore.js bảo: "Mở chat conversation ABC" → sidebar-chat.js xử lý toàn bộ
sidebar-chat.js bảo: "User bấm back" → explore.js hiện lại danh sách

4. Trải nghiệm người dùng (như Telegram thu nhỏ)

User Flow
Bước 1: Mở app
→ Sidebar hiện "GitChat" header + tabs Inbox/Friends/Channels
→ Danh sách conversations hiện ra (avatar, tên, preview tin nhắn cuối)
Bước 2: Click vào 1 conversation
→ Slide animation: danh sách trượt sang trái, chat view trượt vào từ phải
→ Tabs biến mất, header đổi thành: ← Back + Avatar + Tên + Search + Menu
→ Tin nhắn hiện ra, input ở dưới cùng
Bước 3: Chat
→ Gõ tin nhắn, gửi bằng Enter
→ Scroll lên → tự load tin nhắn cũ (infinite scroll)
→ Hover tin nhắn → hiện nút React/Reply/Copy/More
→ Bấm emoji, pin, search, attach file... mọi thứ hoạt động trong sidebar
Bước 4: Quay lại
→ Bấm ← Back
→ Draft tự lưu
→ Slide animation ngược: chat trượt ra phải, danh sách trượt về từ trái
→ Tabs hiện lại, vị trí scroll trong danh sách giữ nguyên

5. Tại sao approach này tốt hơn branch trước?

Branch trước (LỖI)
• Copy 40+ functions từ chat.js
• State phân mảnh (3 biến cho 1 concept)
• Scroll timer conflict nhau
• Mỗi subagent code 1 kiểu
• 5000+ dòng trong 1 file
Branch này (MỚI)
• Viết mới từ đầu cho 300px
• 1 state object duy nhất
• 1 scroll listener duy nhất
• 2 file tách biệt, ranh giới rõ
• Mỗi file có 1 nhiệm vụ
