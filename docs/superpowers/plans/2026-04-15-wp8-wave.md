# WP8 Wave / Say Hi — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Ship WP8 Wave end-to-end per corrected spec: Wave is a notification-only ping for non-mutual online users in Discover → Online Now. No message body, no auto-DM, no Wave Inbox screen. DM only materializes when recipient taps the wave notification (responds). Rate-limit 1 per (sender, target) pair, enforced BE-side.

**Spec:** `docs/superpowers/specs/2026-04-15-wp8-wave-design.md`
**BE requirements:** `docs/sync/be-requirements-wave.md`

**Context / correction history:** Earlier iterations in this planning session got the scope wrong twice — first as a pre-canned DM, then as a Wave Inbox screen with Reply/Dismiss. Spec re-read (`2026-04-13-gitchat-rebrand-feature-spec.md §WP8`) corrected the model. The WP6 Profile Card Wave button (shipped 2026-04-14) is **out of spec** and must be removed as part of this plan. Discover → Online Now section already exists in `explore.js` but sources from `chatFriends` (mutuals only — structurally wrong) and has a hard-coded `disabled "Coming soon"` Wave button.

**Architecture:**
- **Sender:** Discover → Online Now row → Wave button → `discover:wave` postMessage → `explore.ts` handler → `apiClient.wave(login)` → `POST /waves` → UI feedback (button → `waved ✓`).
- **Receiver:** Wave noti already rendered by WP10. New: notifications-pane tap handler on wave rows calls `apiClient.waveRespond(wave_id)` → BE creates DM atomically → FE navigates to new DM.
- **Removal:** Profile Card stranger Wave button + `profileCard:wave` message path + `WaveMockStore` + hover-card wave branch all deleted.

**Tech stack:** TS strict, VS Code Webview API, vanilla JS + CSS, `--gs-*` tokens. No framework. Verification per task: `npm run check-types` + `npm run lint` + manual smoke.

**Commit policy:** ≤5 commits per feature (user preference). Plan groups into **4 commits**.

- [ ] **Step 3.2:** Add `wave()` method to `ApiClient` in `src/api/index.ts` after `unfollowUser`:
  ```ts
  async wave(targetLogin: string): Promise<WaveResponse> {
    log(`[API] POST /waves target=${targetLogin}`);
    const { data } = await this._http.post("/waves", { target_login: targetLogin }, { timeout: 10000 });
    const d = data?.data ?? data;
    return { success: !!d?.success, wave_id: d?.wave_id ?? d?.id ?? "" };
  }
  ```

- [ ] **Step 3.3:** Add `waveRespond()` method (used by Task 4):
  ```ts
  async waveRespond(waveId: string): Promise<WaveRespondResponse> {
    log(`[API] POST /waves/${waveId}/respond`);
    const { data } = await this._http.post(`/waves/${waveId}/respond`, {}, { timeout: 10000 });
    const d = data?.data ?? data;
    return { conversation_id: d?.conversation_id ?? d?.id ?? "" };
  }
  ```
  **Fallback note:** if BE lacks this endpoint, Task 4 will use `createConversation(sender_login)` instead. Keep this method around as no-op dead code until BE confirms — or delete if BE says no.

- [ ] **Step 3.4:** (Optional — conditional on BE) Add `getDiscoverOnlineNow()`:
  ```ts
  async getDiscoverOnlineNow(): Promise<{ login: string; name: string | null; avatar_url: string | null }[]> {
    const { data } = await this._http.get("/discover/online-now");
    const d = data?.data ?? data;
    return Array.isArray(d?.users) ? d.users : (Array.isArray(d) ? d : []);
  }
  ```
  If BE declines, skip this step.

- [ ] **Step 3.5:** In `src/webviews/explore.ts`, add the handler for `discover:wave` in `onMessage` switch (place near existing discover handlers):
  ```ts
  case "discover:wave": {
    const login = (msg.payload as { login: string }).login;
    try {
      const result = await apiClient.wave(login);
      if (!result.success) {
        this.view?.webview.postMessage({ type: "discoverWaveResult", login, success: false });
        vscode.window.showErrorMessage(`Couldn't wave at @${login}.`);
        break;
      }
      vscode.window.showInformationMessage(`Waved at @${login} 👋`);
      this.view?.webview.postMessage({ type: "discoverWaveResult", login, success: true });
    } catch (err) {
      const status = (err as { response?: { status?: number } })?.response?.status;
      if (status === 403) {
        // already_waved / mutual / blocked — treat as terminal waved state from UI POV
        this.view?.webview.postMessage({ type: "discoverWaveResult", login, success: true });
      } else {
        this.view?.webview.postMessage({ type: "discoverWaveResult", login, success: false });
        vscode.window.showErrorMessage(`Failed to wave at @${login}. Please try again.`);
      }
    }
    break;
  }
  ```

    var waveId = (notif.metadata && notif.metadata.wave_id) || notif.id;
    var sender = notif.actor_login;
    if (waveId && sender) {

      });

- [ ] **Step 4.2:** In `src/webviews/explore.ts`, add the handler:
  ```ts
  case "notifications:waveRespond": {
    const { wave_id, sender_login, notif_id } = msg.payload as {
      wave_id: string; sender_login: string; notif_id: string;
    };
    try {
      let conversationId: string;
      try {
        const result = await apiClient.waveRespond(wave_id);
        conversationId = result.conversation_id;
      } catch (err) {
        // Fallback: BE doesn't have /waves/:id/respond. Use createConversation.
        const status = (err as { response?: { status?: number } })?.response?.status;
        if (status === 404 || status === 405) {
          log(`[wave] waveRespond not available, falling back to createConversation`);
          const conv = await apiClient.createConversation(sender_login);
          conversationId = conv.id;
        } else {
          throw err;
        }
      }
      // Mark the wave notification as read
      if (notif_id) { await apiClient.markNotificationsRead([notif_id]); }
      // Navigate to the DM conversation
      await vscode.commands.executeCommand("gitchat.openConversation", conversationId);
    } catch (err) {
      vscode.window.showErrorMessage(`Couldn't open wave reply. ${(err as Error).message ?? ""}`);
    }
    break;
  }
  ```
  Verify `gitchat.openConversation` command exists; if not, use `sidebar-chat` postMessage or whatever sidebar-chat uses to focus a conversation.
