> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Redesign the Noti tab to align with the design system, match the styling of other tabs (Chat/Friends/Discover), and fix mention jump-to-message with flash animation.

**Files:**

private _pendingJump: { conversationId: string; messageId: string; timer: ReturnType<typeof setTimeout> } | null = null;

- [ ] **Step 2: Fix notificationClicked to support jump**

Replace the `notificationClicked` handler:

```typescript
case "notificationClicked": {
  const id = (msg.payload as Record<string, string>)?.id;
  if (!id) { break; }
  const notif = notificationStore.items.find((n) => n.id === id);
  if (!notif) { break; }
  await notificationStore.markRead([id]);
  this.refreshNotifications();

  const meta = notif.metadata ?? {};
  if (meta.conversationId) {
    const messageId = meta.messageId as string | undefined;
    if (messageId) {
      if (this._activeChatConvId === meta.conversationId) {
        // Already open — jump directly
        this.postToWebview({ type: "chat:jumpToMessage", messageId });
      } else {
        // Set pending jump — will fire after chat:ready
        if (this._pendingJump) { clearTimeout(this._pendingJump.timer); }
        const timer = setTimeout(() => { this._pendingJump = null; }, 3000);
        this._pendingJump = { conversationId: meta.conversationId, messageId, timer };
      }
    }
    await this.navigateToChat(meta.conversationId);
  } else if (notif.type === "follow") {
    this.postToWebview({ type: "showProfileCard", login: notif.actor_login });
  } else if (meta.url) {
    vscode.env.openExternal(vscode.Uri.parse(meta.url));
  }
  break;
}
```

- [ ] **Step 3: Add chat:ready handler in chat prefix router**

In the `if (msg.type.startsWith("chat:"))` block, add **before** the `reloadConversation` check:

```typescript
// chat:ready — fired by sidebar-chat.js after render, used for deferred jump-to-message
if (chatType === "ready") {
  const readyConvId = (msg as unknown as Record<string, string>).conversationId;
  if (this._pendingJump && readyConvId && this._pendingJump.conversationId === readyConvId) {
    const { messageId, timer } = this._pendingJump;
    clearTimeout(timer);
    this._pendingJump = null;
    this.postToWebview({ type: "chat:jumpToMessage", messageId });
  }
  return;
}
```

- [ ] **Step 4: Emit `chat:ready` from sidebar-chat.js**

sidebar-chat.js does NOT currently emit `chat:ready`. Add the emission at the end of `case 'init'` and `case 'refresh'` handlers in `handleMessage`, after messages are rendered:

```javascript
// Signal chat is ready for deferred actions (e.g. jump-to-message from notifications)
if (vscode) {
  vscode.postMessage({ type: "chat:ready", conversationId: _state.conversationId });
}
```

Add this at the end of `case 'init':` (after `renderMessages` completes) and at the end of `case 'refresh':` (after re-render completes).

- [ ] **Step 5: Add `case 'jumpToMessage'` handler in sidebar-chat.js**

In `media/webview/sidebar-chat.js`, in the `handleMessage` switch, add a new case. The `handleMessage` function strips `chat:` prefix (line 4050), so host's `chat:jumpToMessage` arrives as `jumpToMessage`. Add this case:

```javascript
case 'jumpToMessage': {
  // Host sends messageId after chat:ready for deferred jump (e.g. from notification tap)
  var jumpMsgId = data.messageId;
  if (jumpMsgId) {
    doAction("chat:jumpToMessage", { messageId: jumpMsgId });
  }
  break;
}
```

This creates the round-trip: host → webview (`chat:jumpToMessage` stripped to `jumpToMessage`) → webview calls `doAction("chat:jumpToMessage")` back to host → host `chat-handlers.ts` fetches message context → sends `jumpToMessageResult` back → webview renders + flash.
